Chapter 6: Adding images to html pages

Here we will learn good practices for adding images to the web page. They are added using the <img> tag. The essential attribute is 'src', which is like 'href' for links. Other attributes include:

  • alt: use this to add descriptive text that will be printed if the image source is corrupted.
  • title: give a unique name/id to the image that is not displayed to the user but can be used for backend applications.
  • height and width: exact pixel values can be provided for how much space the image will take. This is considered good practice for webpages where periodic ads apprear. Giving a fixed height and width will prevent the image for being shifted on the webpage due to a pop-up ad.
  • loading: This attribute determines how quickly an image should be loaded on the webpage. By default, this is set to 'eager', which means the image will be loaded well before the user scrolls down the fold to the image. When set to 'lazy', image will only load just before the user is going to scroll below the fold to it. This setting is good for performance optimisation.
  • Using the figure tag indents the image on the webpage. You can add caption underneath the image within the figure tag, using the figcaption tag.

Forms and user-input data in HTML

Contact me

Here is an example of setting up an HTML form with contact information. You use the <form> tag to achieve this.
  • The action attribute of form determines where the user-input data will go. This can be another website or the local host server.
  • You also need to declare what kind of http request is being made (get or post)
  • Within the form, you can declare labels for what type of information needs to be input by the user. <label> tags used for this. The text between the opening and closing tags will be displayed to the user. One of the attributes for label is 'for', where you can declare an id that is used to link the user input via the input tag, to the label tag.
  • To create a section for the user to input data, use the <input> tag. This tag has three essential attributes:
    1. type - declare the type of data to be input by the user.
    2. name - declare a name for this input data. It MUST match the 'for' id of the label the data links to.
    3. id - this can be the same as the string used in 'name' attribute. Id is used by screen readers.
  • There are other attributes that can be used with input tag, but are not essential:
    1. placeholder - declare a string which will be displayed as placeholder to assist the user. It disappears once user inputs something.
    2. autocomplete - Either 'on' or 'off' - HTML will suggest previous entries made in forms.
    3. required - adding this attribute means the input is essential in the form and cannot progress without it. Writing required sets boolean to true. Not writing 'required' has it set to false.
    4. autofocus - when the page is loaded, it will go to the element with autofocus enabled. ONLY one element per page can have the autofocus attribute.
  • What if you want to give the user options to choose from, rather than inputting certain data? This is achieved using the <label> with <select>, rather than <input>. Each option for the user is declared using the <option> tag. Select have essential attributes 'name' and 'id', like input, that need to match the 'for' attribute of label.
  • For semantics and better organisation, the options can be further grouped into option groups. This is done by placing the <optgroup> tags around a set of options.
  • Other attributes of 'select' that can be used are:
    • multiple - writing this in will allow the user to select more than one option.
    • size - this takes a number input which determines how many options will be displayed on the screen before use of scroll wheel is needed. By default, it is set to 1.
  • Different input form labels can be organised into sections using the <fieldset> tag. Use the legend tag to give the fieldset a description.
  • input tag ids MUST be unique across the entire webpage! No two input locations on the same webpage should use the same id, as that will cause errors in the output.
  • How to submit the input data in the form? Need a button with the data type 'submit'. This will create a json from all the fields completed and left blank by the user.
  • The input 'ids' are the dictionary keys, and the user input are the values.
  • A http get request was used in the example form below. Its limitation is quite apparent! Every bit of user input data is added to the url. This is not ideal for security reasons! Must use post request.
